Design Notes

Selection model
Grey ring always visible as affordance. Purple dot + card border on selection. Purple is purely a UI/interaction color — distinct from all health colors.


Health color roles
Left bar, badge text, health dot glow. Card background stays neutral cool-dark regardless of health state. Health color never bleeds into card background.


Task accent
Cyan mid on task bar left border, icons, progress bar. Cyan neon on task %. These are the only two cyan uses — all other elements use grey text hierarchy.

Card anatomy (top → bottom)
1. Header: selector · drive name · health badge · serial
2. Identity row: type · capacity · bus · speed (immutable)
3. Status row: temp · /dev/path · mount status (live)
4. Task zone: name · progress · status text (expandable)
5. Footer: SMART stats · stacked IO


Leader line
Zero-height warm-white gradient underline of identity row. Bridges the visual gap between left content and right-aligned badge/serial. Decorative only.
